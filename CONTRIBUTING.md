# Contributing

## Branching
- Branch from `main`
- Branch name: `type/short-topic` (example: `docs/weekly-update-template`)

## Commits
Use short conventional commits:
- `feat:` new capability
- `docs:` docs updates
- `fix:` corrections
- `chore:` maintenance

## Pull requests
- Link issue (`Closes #...`)
- Keep PR focused and small
- Update docs if behavior/process changed

## Definition of done
- [ ] Issue linked
- [ ] Files are clear and minimal
- [ ] Update note added when relevant
